Multiple AWS-compatible services are launching within 24 hours once billing tests complete. QStorage provides S3-compatible object storage with event notifications support. All services integrate with Quilibrium's privacy primitives and support the same SDK interfaces developers already know.


✅ True MPC: Why It Matters

Cassandra explained how QKMS differs from most "multi-party wallet" services on the market. Many competitors claim MPC but actually generate the full key in one place, then split it using Shamir's secret sharing. When signing, they recombine key shares into one location (either user device or trusted enclave). This means the full key exists somewhere at some point - making whoever holds that moment the custodian.

Quilibrium's approach uses true threshold MPC. Key shares are generated through DKL-S18 (for 2-of-2) or higher threshold variants (for 3-of-5, etc.), and they never exist in the same place. Signing uses oblivious transfer protocols where the key material is never combined. The system supports multiple curves including SECP256K1 (Bitcoin/Ethereum), ED25519 (Solana/Farcaster), and DCAF-448 (Quilibrium stealth signing).

This approach is truly non-custodial and compliant with regulations like those from the New York Department of Financial Services.


✅ Quark SDK Demo

A demonstration showed the Quark SDK enabling tokenized game assets. The demo featured 3D avatar assets stored on QStorage with RDF schema validation and in-game currency backed by Quilibrium token primitives. Users can purchase and own avatars as private tokens, with asset ownership being completely private - the game cannot see your wallet or what you own. The system uses view and spend keys visible only to the user, not the game.

This enables privacy-preserving digital ownership for games and virtual worlds - unlike public blockchain approaches where wallet contents are visible to everyone.
